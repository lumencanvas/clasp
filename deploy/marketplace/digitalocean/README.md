# CLASP Relay -- DigitalOcean Marketplace 1-Click App

Pre-built DigitalOcean Droplet image with the CLASP relay server, Caddy reverse proxy, and an interactive setup script. Users create a Droplet from the Marketplace, SSH in, run `clasp-setup`, and get a production relay in under two minutes.

## What's in the image

- Ubuntu 22.04 LTS
- Docker Engine + Compose plugin
- CLASP relay container image (pre-pulled)
- Caddy 2 container image (pre-pulled)
- `clasp-setup` interactive configuration script
- UFW firewall (SSH, HTTP/S, MQTT, OSC)

## First-boot experience

```
ssh root@<droplet-ip>

  ================================================================
   CLASP Relay
   Real-time infrastructure for connected devices
  ================================================================

   Get started:    clasp-setup
   Documentation:  https://docs.clasp.to

   Status: NOT CONFIGURED
   Run:    clasp-setup
```

Running `clasp-setup` prompts for:

1. Deployment profile (Standard / Full / Minimal)
2. Domain name and Let's Encrypt email
3. Optional feature toggles (MQTT, OSC, rules engine)
4. CORS origins

The script generates `/opt/clasp/docker-compose.yml` and `/opt/clasp/Caddyfile`, starts the containers, and prints the connection URL and admin token.

## Deployment profiles

| Profile | Features |
|---------|----------|
| **Standard** | WebSocket + Caddy TLS + auth + persistence |
| **Full** | Standard + MQTT bridge + OSC bridge + rules engine + metrics |
| **Minimal** | WebSocket only, no TLS (testing or behind another proxy) |

## Building the image

Prerequisites: [Packer](https://www.packer.io/) 1.9+ and a DigitalOcean API token.

```bash
export DIGITALOCEAN_TOKEN="your-api-token"

cd deploy/marketplace/digitalocean
packer init .
packer build template.pkr.hcl
```

This creates a snapshot in your DO account. The snapshot is replicated to all specified regions.

Override defaults:

```bash
packer build \
  -var "relay_version=4.3.0" \
  -var "region=fra1" \
  template.pkr.hcl
```

## Submitting to the Marketplace

1. Build the image with Packer
2. Test by creating a Droplet from the snapshot
3. SSH in and run `clasp-setup` to verify the full flow
4. Submit at https://marketplace.digitalocean.com/vendors
5. Provide the snapshot ID and the listing metadata below

### Listing metadata

- **Name:** CLASP Relay
- **Summary:** Real-time infrastructure for connected devices. State sync, protocol bridging, E2E encryption, and automation in one server.
- **Category:** Developer Tools
- **Minimum Droplet:** Basic, 1 vCPU, 1 GB RAM ($6/mo)
- **Recommended Droplet:** Basic, 1 vCPU, 2 GB RAM ($12/mo)
- **Support URL:** https://github.com/lumencanvas/clasp/issues
- **Documentation URL:** https://docs.clasp.to/deployment/docker

## Ports

| Port | Protocol | Purpose |
|------|----------|---------|
| 22 | TCP | SSH |
| 80 | TCP | HTTP (Caddy redirect to HTTPS) |
| 443 | TCP/UDP | HTTPS + HTTP/3 (Caddy TLS termination) |
| 1883 | TCP | MQTT bridge (optional, Full profile) |
| 8000 | UDP | OSC bridge (optional, Full profile) |

Ports 7330 (WebSocket), 7350 (auth), and 7360 (health) are internal to Docker and proxied through Caddy.

## File layout on the Droplet

```
/usr/local/bin/clasp-setup          # Interactive setup script
/opt/clasp/
  docker-compose.yml                # Generated by clasp-setup
  Caddyfile                         # Generated by clasp-setup
  docker-compose.yml.tpl            # Template (reference)
  Caddyfile.tpl                     # Template (reference)
/var/lib/clasp/
  admin.token                       # Admin CPSK token
  auth.db                           # User auth database
  state.json                        # State snapshots
  rules.json                        # Rules engine config
/var/lib/digitalocean/
  application.info                  # Marketplace metadata
/etc/update-motd.d/
  99-one-click                      # SSH login banner
```

## Updating the relay

```bash
docker compose -f /opt/clasp/docker-compose.yml pull
docker compose -f /opt/clasp/docker-compose.yml up -d
```

Or re-run `clasp-setup` to reconfigure and restart.
