# DigitalOcean 1-Click Deploy

The CLASP Relay is available as a 1-Click App on the DigitalOcean Marketplace. Create a Droplet, SSH in, run one command, and you have a production relay with TLS, auth, and persistence.

## Quick Start

1. Create a Droplet from the CLASP Relay listing in the [DigitalOcean Marketplace](https://marketplace.digitalocean.com)
2. Point your domain's DNS A record to the Droplet IP
3. SSH in:

```bash
ssh root@your-droplet-ip
```

4. Run the setup:

```bash
clasp-setup
```

The setup script walks you through profile selection, domain configuration, and feature toggles. It generates the Docker Compose file, starts Caddy (TLS) and the relay, and prints your connection URL and admin token.

## Deployment Profiles

| Profile | What you get |
|---------|-------------|
| **Standard** | WebSocket + Caddy TLS + auth + persistence + health checks |
| **Full** | Standard + MQTT + OSC + journal + rules + metrics + entity registry + capability tokens |
| **Minimal** | WebSocket only, no TLS, no auth (dev or behind another proxy) |

## Configuration Sections

After picking a profile, the setup script walks through these sections. Every question has a default, so you can press Enter through for a working deployment.

- **Core**: domain, email, CORS, server name, WebSocket port
- **Protocol Bridges**: MQTT port/namespace, OSC port/namespace
- **Security**: auth port, token TTL, capability tokens, entity registry
- **Persistence**: state snapshots, journal backend (SQLite or DefraDB), DefraDB URL
- **Automation**: rules engine JSON, LensVM transforms JSON, app config JSON
- **Federation**: hub URL, router ID, owned namespaces
- **Advanced** (opt-in): session limits, TTLs, QUIC, metrics, rendezvous, drain timeout

## What's Installed

The Droplet image includes:

- Ubuntu 22.04 LTS
- Docker Engine + Compose
- CLASP relay container image (pre-pulled)
- Caddy 2 reverse proxy (pre-pulled)
- `clasp-setup` interactive setup script
- UFW firewall (SSH, HTTP/S, MQTT, OSC enabled)

## Ports

| Port | Protocol | Purpose |
|------|----------|---------|
| 22 | TCP | SSH |
| 80 | TCP | HTTP (Caddy, redirects to HTTPS) |
| 443 | TCP+UDP | HTTPS + HTTP/3 (Caddy TLS termination) |
| 1883 | TCP | MQTT bridge (Full profile) |
| 8000 | UDP | OSC bridge (Full profile) |

WebSocket (7330), auth (7350), and health (7360) are internal to Docker and proxied through Caddy.

## File Layout

```
/usr/local/bin/clasp-setup          Interactive setup (re-run to reconfigure)
/opt/clasp/docker-compose.yml       Generated by clasp-setup
/opt/clasp/Caddyfile                Generated by clasp-setup
/var/lib/clasp/admin.token          Admin CPSK token
/var/lib/clasp/auth.db              User auth database
/var/lib/clasp/state.json           State snapshots
/var/lib/clasp/journal.db           Journal (if enabled)
/var/lib/clasp/rules.json           Rules engine config (if enabled)
/var/lib/clasp/lenses.json          LensVM config (if enabled)
```

## Connecting

After setup completes:

```javascript
import { Clasp } from '@clasp-to/core'

const clasp = new Clasp('wss://relay.example.com')
await clasp.connect()

clasp.set('/lights/brightness', 0.8)
```

With auth:

```javascript
const clasp = new Clasp('wss://relay.example.com', {
  token: 'your-token-here'
})
```

## Managing the Relay

```bash
# View logs
docker compose -f /opt/clasp/docker-compose.yml logs -f

# Restart
docker compose -f /opt/clasp/docker-compose.yml restart

# Stop
docker compose -f /opt/clasp/docker-compose.yml down

# Update to latest image
docker compose -f /opt/clasp/docker-compose.yml pull
docker compose -f /opt/clasp/docker-compose.yml up -d

# Reconfigure
clasp-setup
```

## Building the Image Yourself

If you want to build a custom marketplace image instead of using the published one:

```bash
cd deploy/marketplace/digitalocean
export DIGITALOCEAN_TOKEN="your-token"
packer init .
packer build template.pkr.hcl
```

This creates a snapshot in your DigitalOcean account. Create Droplets from it via the API or console.
